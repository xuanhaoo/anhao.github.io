为什么需要Java内存模型 　　Java内存模型（Java Memory Model：JMM）是Java虚拟机规范定义的，用来屏蔽Java程序在不同的硬件环境和操作系统对内存访问的差异性，使之Java程序在各种不同的平台上达到内存访问的一致性。 CPU和缓存一致性 　　计算机在执行程序的时候，每条指令都是放在CPU中进行执行的，而执行的时候一般都会与数据交互，但是数据又是存放在计算机主内存（计算机物理内存）中的，最开始的单核的CPU就处理问题很简单，直接读取然后放回主内存即可，但是随着计算机技术的不断发展，CPU的执行速度越来越快，从内存中读取和写入数据的速度和CPU的执行速度差距越来越大，就会导致CPU每次操作内存都会花费很多等待时间，造成资源的浪费。 　　后来人们提出一种高速缓存交互方案，就是在CPU和主内存之间增加高速缓存，这样当程序运行时，会将需要的数据从主内存中拷贝一份在高速缓存中，CPU在计算数据时直接从高速缓存中读取，当处理完数据再将数据刷新回主内存。 　　CPU技术还在不断发展，一层缓存满足不了需求，于是衍生出多级缓存。CPU的多 ...
