Java之单例模式笔记

发表于2018-12-01|更新于2021-01-20|技术
|字数总计:1.2k|阅读时长:4分钟|阅读量:

单例模式概述

 　　单例模式是一种常见的设计模式，最近也再一次进行学习，所以写一篇日志记录梳理一下。
 　　单例模式的构造方法私有，确保该类只能由自己初始化，进而向其他的类提供这一唯一实例，减少资源的浪费，系统的开销。单例模式应用场景比如数据库的连接配置，日志对象等信息可以使用单例作为载体。一个单例模式具有以下3个特点：

  • 一个类只能有一个实例对象的存在
  • 该实例对象由自己创建唯一实例
  • 该类要向其他的类提供这一实例对象

实现方式

 　　了解了单例模式的基本概述后，一起来看一下模式的实现，单例模式有多种实现，我这里主要记录3种方式：饿汉式，懒汉式，枚举实现。

懒汉式

1.懒汉式普通写法，这种实现方式在单线程环境下是没有线程安全问题的，但在多线程环境下，同时调用getInstance方法，就有可能创建多个对象。

2.懒汉式的线程安全写法

 　　该实现方法是线程安全的，但要注意一点，单例对象必须要用volatile关键字修饰。在多线程环境下，假如当A和B两个线程执行getInstance方法，先是A获取到锁执行到24行了，此时，CPU指令重排执行顺序可能变成了132，然后B执行方法，判断对象是否为空，不为空，返回对象，于是，错误就发生了。所以需要使用volatile来防止发生指令重排。

volatile关键字：

  • 保证变量的可见性：一个线程更新了变量的值，会在写操作后加入store内存屏障指令（CPU指令），将本地内存中的值刷新到主内存；在读取变量操作前，会加入一个load指令去主内存中读取变量最新的值；
  • 防止指令的重排序：在volatile读或写操作时，都会加入屏障指令，防止指令重排序；

饿汉模式

1.静态方法实现：单例类在类装载的时候初始化，不延迟加载对象。

静态块的实现方式：这种方式实现时要注意静态块的位置，要放在单例对象定义之后。

枚举实现

 　　枚举类就是一个单例模式的实现，并且实现了序列化，既保证了延时加载，有保证了线程的安全。所以可以使用枚举类来实现单例模式。

总结

 　　单例模式的实现还有很多，就不一一列举了，大家有兴趣的话可以查阅资料。我们根据适合的场景选择合适的单例实现方式。
